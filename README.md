# Make.com Onboarding Experience — Product Teardown

This repository contains a product teardown analyzing the onboarding experience of **Make.com**, a visual automation platform.

The goal of this analysis is to understand how Make guides new users from sign-up to building their **first automation**, identify friction points in the onboarding journey, and propose product improvements.

---

## Overview

Make.com enables users to connect applications and automate workflows without coding. 

This teardown focuses on:

- How new users are onboarded
- Whether onboarding helps users activate successfully
- Where users may face confusion or friction
- Opportunities to improve the first-time user experience

---

## Key Areas Analyzed

### 1. Target Users & Personas
Identification of primary users including:
- Growth marketers
- Operations managers
- Product teams
- No-code builders

### 2. Onboarding Flow
Breakdown of the onboarding steps including:

- Account creation
- Discovery questions
- Automation experience level
- Company size
- Job role
- Tech stack selection
- Automation goals

### 3. Identified Product Frictions

**Pain Point 1 — Survey Data Not Used**

Make collects onboarding responses but does not use them to personalize the first dashboard experience.

**Pain Point 2 — First Automation Decision is Unclear**

New users are asked to choose between:
- Build from scratch
- Start with a template

Without understanding automation concepts yet.

**Pain Point 3 — 'Operations' Metric is Confusing**

The dashboard shows an operations usage counter but does not clearly explain:
- What counts as an operation
- How operations are consumed
- How workflows impact usage limits.

---

## Product Strengths

Despite onboarding friction, Make provides powerful capabilities:

- Visual drag-and-drop automation builder
- Support for 3000+ integrations
- Flexible automation logic
- Strong template ecosystem

---

## Proposed Improvements

Some potential product improvements include:

- Personalizing the dashboard based on onboarding responses
- Guiding users through their first automation scenario
- Recommending starter templates based on selected tools
- Explaining key usage metrics directly within the dashboard

---

## Metrics to Track

To measure onboarding success, product teams could track:

- Activation rate (users creating first automation)
- Time to first automation
- Template usage rate
- Onboarding drop-off rate
